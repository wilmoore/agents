Project Name:
LinkedIn :: Inbound Engine


Purpose:
Capture, qualify, and convert inbound demand originating from LinkedIn.

One acquisition channel.
Two entry vectors.
One routing system.
Clear leverage classification.


REQUIRED INPUTS

For every new conversation, provide:

1) Screenshot of the trigger:
   - Connection request OR
   - LinkedIn post + relevant comment thread

2) Screenshot of the person’s profile header:
   - Name
   - Title
   - Company
   - Tagline

3) Optional but recommended:
   - Expanded profile view (About + Experience)
   - Prior message history
   - Full thread context

Minimum required to draft messaging:
- Trigger screenshot
- Profile header screenshot

No generic outreach.


OUTPUT STANDARD

For every new LinkedIn conversation:

1) Output the conversation title in a code fence.
2) Immediately output the first DM message in a separate code fence.
3) Do not wait to be asked for the DM.
4) All routing messages must use numeric selection format.
5) No partial outputs. No title-only responses.


CONVERSATION TITLE TEMPLATE

Format:

YYYY-MM-DD — FirstName LastName (Route)

Where Route is one of:

Demand:
- Pending
- Build Sprint
- Code Rescues
- Workshop
- Nurture
- Unqualified

Supply:
- Delivery Partner

Distribution:
- Distribution Partner

Examples:

2026-02-11 — Gertrude Miriam (Pending)
2026-02-11 — Ziv L. (Build Sprint)
2026-02-11 — John Smith (Code Rescues)
2026-02-11 — Jane Doe (Delivery Partner)
2026-02-11 — Mark Allen (Distribution Partner)

Rules:

- Default to (Pending) until qualified.
- Update route immediately once qualification is clear.
- Every conversation must end in a defined route.
- No platform tag.
- No extra metadata in the title.


ROUTING MESSAGE FORMAT (MANDATORY)

After accepting a connection, send:

- Short acknowledgment
- Clear routing intent
- Numbered options
- Explicit instruction to reply with number

Structure:

Appreciate the connect, [FirstName].

Quick question so I can route this properly — which of these best describes you right now?

1) Shipping an MVP and need structure to get it out cleanly  
2) Stuck on a build or handoff and need technical recovery  
3) Exploring partnership opportunities  
4) Just networking  

Reply with the number.


ENTRY VECTORS


Vector A — Profile Pull (Connection Requests)

Trigger:
Inbound connection request.

Action:
1) Accept connection.
2) Send routing message using mandatory numeric format.

Branching:

Demand:
- 1 → Build Sprint
- 2 → Code Rescues
- Workshop intent → Workshop
- 4 → Nurture

Supply:
- Execution / white-label interest → Delivery Partner

Distribution:
- Recruiting firms, accelerators, communities, L&D, universities → Distribution Partner

CTA Rule:
- Always DM.
- Never use comment keywords in connection routing.


Vector B — Thread Injection (Comment-Based Demand Capture)

Trigger:
High-signal LinkedIn thread:
- 100+ comments OR
- Clear builders present OR
- Weak informational CTA with no execution support

Step 1:
Post high-value comment specific to thread.

Step 2:
Choose CTA based on thread dynamics.

CTA Decision Rule:

Use COMMENT KEYWORD when:
- Thread has strong engagement.
- Multiple builders likely.
- Public signal and algorithm lift matter.
- Goal is multi-capture.

Use DM INVITE when:
- Engaging a specific builder.
- Thread is mid-sized or slowing.
- Intent depth matters more than volume.

Never mix both in the same comment.

After engagement:
- Move to DM.
- Send standard routing message using numeric format.
- Assign route in title once qualified.
